# ⏳ TimeMachine

**A live-style tracker for the wild mammals moving through San Francisco — and a time machine to replay where they've been.**

Centered on **SHACK15** at the Ferry Building. Inspired by the [OCEARCH shark tracker](https://www.ocearch.org/tracker/), built for the great over-land flyways of the city.

</div>

---

## What it is

Five collared Bay Area mammals — a coyote, a raccoon, a striped skunk, a gray fox and a black-tailed deer — with three weeks of telemetry around San Francisco. Open it and you get a dark, full-screen map clustered on SHACK15; click any animal to see its track, its **vitals**, and the two numbers that matter most in the field: **distance walked** and **speed**. Then drag the **time machine** to replay the last three weeks, or hit **LIVE** to watch fresh fixes stream in.

## Features

- 🗺️ **Live map** (Leaflet) with dark / terrain / satellite basemaps — no API keys.
- 🐾 **5 animals**, color-coded tracks, named stopovers, a pulsing head for transmitting collars, and a labeled **SHACK15** landmark pin.
- ⏱️ **Time-travel scrubber** — drag or press play (0.5×–4×) to replay every track over time.
- 🔴 **LIVE mode** — simulated real-time fix stream; animals move and their metrics update on the spot.
- 📊 **Metrics that matter** — **feet walked** (+ miles), **average speed on the move**, **top speed**, and **last-night distance**, computed live from the fixes.
- 🔎 **Filter & search** by species, sex, life stage, transmitter status, or name.
- 🤖 **Ask Claude → Field Notes** — a one-click plain-English read of an animal's recent movements (falls back to a built-in note when no API key is set).
- 📱 Responsive down to phone widths.

## Quick start

It's a static site with **no build step**.

```bash
# easiest: just open it
open index.html          # macOS
start index.html         # Windows
# or serve it (recommended; identical to production):
python -m http.server 8000
# → http://localhost:8000
```

> Map tiles need a network connection; the app shell, tracks and markers work offline (Leaflet is vendored locally).

## The metrics

| Metric | Meaning | Why it matters |
|---|---|---|
| **Feet walked** | Total great-circle distance across all fixes | Home-range size, energy use, habitat quality |
| **Avg speed (on the move)** | Mean per-segment speed while moving (≥0.3 mph) | Behavioral baseline; deviation = stress/disturbance |
| **Top speed** | Fastest GPS-sampled segment | Flight/pursuit events |
| **Last night** | Distance in the most recent ~14 h | Daily activity / "is it healthy" |

A flat-lined speed is a **mortality alert**; a spike is a disturbance or pursuit. See [`DEMO_SCRIPT.md`](DEMO_SCRIPT.md) for the full demo + the government/private use-case pitch.

## Claude Field Notes (optional)

The **Ask Claude** button calls the Anthropic API directly from the browser. To enable live notes, the app prompts once for an API key and stores it in `localStorage` (never committed, never sent anywhere but Anthropic). Without a key it returns a pre-written naturalist note per animal, so the demo always works.

## How it was built

The interface was generated by **8 DeepSeek agents working in parallel** — one per module — against a locked contract ([`SPEC.md`](SPEC.md)), then hardened by a **Claude adversarial validation swarm** (one reviewer per module + an integration reviewer, scored on a 7-point rubric) whose findings were integrated and fixed. The dataset is produced by a reproducible generator.

```
TimeMachine/
├── index.html            # app shell (the DOM contract)
├── styles/main.css       # dark theme
├── scripts/
│   ├── data.js           # window.BIRD_DATA (generated)
│   ├── util.js           # TM helpers + TM.metrics()
│   ├── map.js            # Leaflet map, tracks, markers
│   ├── timeline.js       # the time-travel scrubber
│   ├── live.js           # simulated live fix stream
│   ├── claude.js         # Ask Claude → Field Notes (+ fallback)
│   ├── sidebar.js        # list, filters, profile
│   └── app.js            # orchestrator
├── assets/vendor/leaflet/ # Leaflet 1.9.4 (vendored)
├── tools/generate_data.py # reproducible track generator
├── SPEC.md  DEMO_SCRIPT.md
```

Regenerate the data: `python tools/generate_data.py`

## Data disclaimer

The **species, locations and home-range behavior are real and documented** for the San Francisco Bay Area. The **individual animals, names, collar programs, dates and GPS fixes are synthetic** — this is a demonstration, not a biological data source.
